## Ultrathink decision (package A — Discovery step 0)

- **Option A (selected): insert `- [ ] 0.` before step 1; keep every
  existing step number.** Cross-references to SETUP step numbers exist in
  ADOPTION.md, the profile README, and the seed README — renumbering is
  exactly the reference-rot class finding 3 documents ("steps 1–5" vs 6).
- Option B (rejected): renumber 1–13 — many cross-refs to touch across
  parity-tested files, high drift risk, no added enforcement value.
- Option C (rejected): separate DISCOVERY.md — a new root-singleton doc is
  the guard-gap class v2.5.2 just closed; the checklist is the single home.
- Key rule from the debate: forbid the *premature recommendation*, not just
  add questions (the fisi agent DID ask what the app is — in the same
  question round as the premarked stack answer). Guard both failure
  directions: skipped interview (three anchors: step 0 first, step 2
  "profiles never justify the stack", step 9 "an ADR invented here means
  step 0 was skipped") and interview bloat (minimum catalog, focused
  rounds, explicit delegation escape hatch recorded honestly in the ADR).
- ASK markers stay minimal (header sentence + inline ASK at steps 0, 7,
  11); a DECISION-marker taxonomy was rejected as speculative structure.
- GitHub completion lands as an unnumbered closing note (mirrors the
  existing Cowork note pattern): the run is complete only when origin
  exists, the first push is up, and the ci.yml run came back green.
